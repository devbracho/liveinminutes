Supabase ships with a shared SMTP service for development. It caps out at **2–3 OTP emails per hour** across your entire project. Any real traffic hits this limit immediately — users see *"email rate limit exceeded"* and can't sign in.

The fix is to connect your own email provider. [Resend](https://resend.com) is the fastest option: free up to 3,000 emails/month, and Supabase has a native marketplace integration for it.

## Option A — Resend Marketplace integration (recommended)

This is one click and requires no manual SMTP config.

### 1. Install the integration

1. Open your Supabase project dashboard.
2. Go to **Integrations** (left sidebar) → search for **Resend**.
3. Click **Connect** → follow the OAuth flow to authorize Resend.
4. Supabase automatically sets the SMTP credentials. Done.

### 2. Verify it works

Send a magic link from your app. Check the **Resend dashboard → Emails** tab — you should see a delivery record within seconds.

## Option B — Manual SMTP (Resend, SendGrid, Postmark, etc.)

Use this if you want to use a different provider or need more control.

### 1. Get SMTP credentials from your provider

**Resend:**
1. Create an account at [resend.com](https://resend.com).
2. Go to **API Keys** → Create a key with "Sending access".
3. Your SMTP credentials are:
   - Host: `smtp.resend.com`
   - Port: `465` (SSL) or `587` (TLS)
   - Username: `resend`
   - Password: your API key

**SendGrid:**
1. Create an account at [sendgrid.com](https://sendgrid.com).
2. Go to **Settings → API Keys** → Create a key with "Mail Send" permission.
3. SMTP credentials:
   - Host: `smtp.sendgrid.net`
   - Port: `587`
   - Username: `apikey`
   - Password: your API key

### 2. Add SMTP settings in Supabase

1. Open your Supabase project dashboard.
2. Go to **Project Settings** (gear icon) → **Auth**.
3. Scroll to **SMTP Settings**.
4. Toggle **Enable Custom SMTP** on.
5. Fill in:
   - **Sender name**: e.g. `LiveInMinutes`
   - **Sender email**: e.g. `no-reply@yourdomain.com` (must be verified with your provider)
   - **Host**, **Port**, **Username**, **Password**: from step 1.
6. Click **Save**.

### 3. Verify your sender domain (required)

Your email provider requires that the sender domain (`yourdomain.com`) is verified via DNS records. Follow the provider's verification guide — it usually takes 5–10 minutes to propagate.

If you skip this, emails will bounce or land in spam.

### 4. Test the connection

Still in **SMTP Settings**, click **Test SMTP connection**. Supabase sends a test message. Check your inbox.

## Update redirect URLs (both environments)

Magic links redirect to your app's `/auth/callback` URL. Supabase rejects redirects to domains it doesn't recognize.

### Site URL

Set **Site URL** to your production domain — this is the base URL Supabase uses when building magic link emails. If it is still `http://localhost:3000`, every magic link sent to real users will point at your laptop.

```
https://yourdomain.vercel.app
```

Click **Save changes**.

### Redirect URLs

These are the specific URLs Supabase is allowed to redirect to after auth. Add one per environment:

| Environment | URL |
|---|---|
| Local dev | `http://localhost:3000/auth/callback` |
| Vercel previews | `https://*.vercel.app/auth/callback` |
| Production | `https://yourdomain.vercel.app/auth/callback` |

Use the exact `/auth/callback` path — not a wildcard `/**`. It is more secure and matches the `emailRedirectTo` value the app sends.

---

## Set NEXT_PUBLIC_SITE_URL in Vercel

The `emailRedirectTo` URL is built from `NEXT_PUBLIC_SITE_URL`. Make sure it points to your production domain — not `localhost`.

```bash
vercel env add NEXT_PUBLIC_SITE_URL production
# enter: https://yourdomain.com
```

Or set it in the Vercel dashboard: **Project → Settings → Environment Variables**.
